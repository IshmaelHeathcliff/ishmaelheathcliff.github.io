Minecraft红石逻辑基础

可使用如红石计划：集成（Project Red： integration）
等 mod 大大简化原版红石结构

信号类型

  • 持续信号~
    • 111
    • 000
  • 短促信号、脉冲信号^
    • 101
    • 010
  • 边沿信号|
    • 上边沿、上升沿：01
    • 下边沿、下降沿：10

信号发生器

  • 基础信号发生器
    • 红石火把~
    • 红石块~
    • 拉杆~
    • 按钮^
  • 传感器
    • 光照传感器~
    • 雨雪传感器~
    • 压力板~^
    • 比较器容器检测~
  • 脉冲信号发生器^
  • 随机信号发生器~^

特殊单元

  • 中继器
    • 强充能
    • 信号加强
    • 信号延迟
    • 锁存
  • 比较器
    • 容器检测
    • 信号比较
    • 信号作差

基础逻辑结构

  • 非门 NOT：反转信号
  • 与门 AND：都为 1 时为 1，否则为 0
  • 或门 OR：都为 0 时为 0，否则为 1
  • 与非门 NAND：都为 1 时为 0，否则为 1
  • 或非门 NOR：都为 0 时为 1，否则为 0
  • 异或门 XOR：相同时为 0，不同时为 1
  • 异或非门 XNOR（同或门）：相同时为 1，不同时为 0

进阶结构

锁存器与触发器

锁存器（Latch）和触发器（Flip-Flop）为记忆电路，可存储输入信号，将短促信号转变为持续信号。锁存器为电平触发装置，检测信号状态（1 为高电平，0 为低电平）并储存。触发器为边沿触发装置，检测信号变化（边沿信号，0 变 1 为上边沿信号，1 变 0 为下边沿信号）从而触发状态改变。锁存器比触发器更简单和基础，锁存器可扩展为触发器。其中 RS 或非锁存器为基本单元，又称为 RS 基本触发器。

RS 锁存器（RS 或非锁定器，Reset-Set）

  • 输入端 S 和 R
  • 输出端 Q 和
  • S 输入 1 后，则 Q 为 1， 为 0
  • R 输入 1 后，则 Q 为 0， 为 1
  • 禁止 S 和 R 同时输入 1

稳定器
  • 只有 S 和 Q 的 RS 锁存器
  • 将短促信号转变为永久信号

边沿触发器

  • 上边沿触发器：将上边沿信号转变为一次短促信号
  • 下边沿触发器：将下边沿信号转变为一次短促信号
  • 将锁存器改造为触发器

  • 根据两个中继器的延迟差来调整上边沿还是下边沿触发
  • 延迟差至少为 2tick

D 触发器（Data）

  • 数据输入端 D 和时钟信号输入端 C
  • 信号强制端 SR
  • 输出端 Q 和
  • C 端输入上边沿信号时，将 D 锁存到 Q
D 锁存器
  • C 为 0 时，Q 与 D 一致
  • C 为 1 时，保持 Q 不变，与 D 无关

JK 触发器

解决 RS 锁存器中 R 和 S 不能同时为 1 的问题，同时为 1 时信号将反转。并且由时钟信号来控制锁存。

J K Q(t)
0 0 Q(t-1)
0 1 0
1 0 1
1 1 (t-1)

T 触发器（Toggle）

  • 接受边沿信号改变输出状态
  • 将两次脉冲信号合成为一个

同步器

当两端各自有过一次上边沿信号后，输出一个脉冲信号。将两个 RS 锁存器的输出通过与门连接，将两个 S 端作为输入，将输出信号回传到 R 端重置。通过边沿信号触发器接受信号。
