标签系统

标签的应用

  • 标签的目的是快速检索
  • 标签用于揭示场景和主题，在一个宽泛的角度来进行概括
  • 标签不应该作用于一个具体的对象，而应该作用于一个抽象或主题
  • 标签也可以用于状态标记
  • 标签的总数量应该处于控制下
  • 一个应用足够广泛的主题才值得新建一个标签
  • 标签一方面使用在 page 的 Front-matter 中对 page 进行分类
  • 另一方面使用在 Daily Note 中对零散内容进行主题标记

标签的分类

  • 标签的使用可分为多个维度
    • 话题标签
    • 状态标签
    • 分类标签

标签与双链区别

  • 双链的目的是将不同页面进行关联以揭示话题间的内在联系
  • 这种联系将帮助进一步的整理和思考
  • 双链应该连接一个已有的或可能有的页面，这个页面应该包含一定的内容来解释一个特定的话题
  • 双链也可代替话题标签，来减轻标签系统的复杂度

Daily Note 使用场景

以 Unity 为例：

如果一段内容只是提到 Unity，并没有太多实际内容，则既不使用标签也不使用双链，直接通过检索 Unity 来得到这些内容。

如果一段内容是对使用 Unity 进行的项目的描述，但不直接涉及 Unity 的使用方法，则可以使用标签来标注场景。

如果一段内容是对 Unity 的 API、插件、模块等的具体描述，应使用双链连接该具体描述的页面，并将该页面添加 Unity 标签。

Daily
Note 中应该减少对 Unity 这种宽泛缺乏具体内容的词进行直接的双链连接，而更多的使用双链来作为 page 的创建入口

Page 使用场景

  • page 中应该尽量少使用标签，而使用双链来连接其他话题
